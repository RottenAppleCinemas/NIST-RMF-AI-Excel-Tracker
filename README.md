# NIST-RMR-AI-Excel-Tracker


NIST AI RMF Organizational Profile Template
Based on the NIST AI Risk Management Framework 1.0 (NIST AI 100-1) and its companion Playbook

What is an Organizational Profile?
An AI RMF Profile describes how an organization currently manages AI risk ("Current Profile") and the outcomes it wants to achieve ("Target Profile"), using the GOVERN, MAP, MEASURE, and MANAGE functions, categories, and subcategories of the AI RMF Core. Comparing the two profiles reveals gaps that can be used to build a prioritized action plan / roadmap.

How this workbook is organized
"1. Instructions (this sheet)
2. Rating Scale - the maturity scale used to score Current and Target profiles, and the Priority/Risk scale
3. GOVERN / MAP / MEASURE / MANAGE - one sheet per AI RMF function, listing every category and subcategory, with side-by-side Current vs. Target rating, an auto-calculated Gap, and fields for notes, owner, and target date
4. Gap Analysis Dashboard - roll-up of gaps by function and category, sorted to highlight the largest gaps, with a chart"

How to complete the profile
"For every subcategory:
  - Current Rating: select from the dropdown how well your organization implements this subcategory TODAY
  - Current Notes: briefly describe current state, evidence, or existing controls
  - Target Rating: select the maturity level you WANT to reach (your target state)
  - Target Notes: describe what 'fully achieved' looks like for your organization
  - Priority: select High / Medium / Low for closing this gap
  - Owner: who is accountable for closing the gap
  - Target Date: when you plan to close the gap
  - Action Plan: the specific steps needed to move from Current to Target
The Gap column, the Gap Analysis Dashboard, and all summary statistics update automatically as you fill in ratings."

Tips
"- Mark subcategories that do not apply to your organization or AI use case as 'N/A' rather than leaving them blank.
- Use the Rating Scale sheet as a shared reference so raters apply the scale consistently.
- Revisit and update the Current Profile periodically - profiles are meant to evolve over time.
- Consider maintaining separate copies of this workbook for different AI systems or use cases if their risk profiles differ significantly.
- This template is a voluntary tool to help apply the AI RMF; it is not an official NIST publication or scoring methodology."
